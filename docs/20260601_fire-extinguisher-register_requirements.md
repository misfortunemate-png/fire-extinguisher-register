## 1. 目的

板橋区内の公共施設（68施設・消火器401本）を巡回する際に、消火器の台帳管理と点検記録をPixel 10上で行えるPWA。

ラベル写真のAI読み取り＋プリセットマッチングで登録を効率化し、不具合のある消火器は看板式報告カードとして画像化、週報xlsxに自動出力する。

## 2. 利用者・利用環境

- 利用者: ショウゴさん（1名）
- 主要デバイス: Pixel 10（Chrome / PWA）
- 利用場面: 現場巡回中（モバイル回線 or オフライン）

## 3. 機能要件

### 3.1 施設一覧（ホーム画面）
- 68施設の一覧表示・検索・充足状態表示
- 台帳CSV出力ボタン、不具合報告xlsx出力ボタン

### 3.2 施設詳細画面
- 消火器一覧、プリセットボタン（組み込み＋カスタム＋手動入力）
- 消火器カード（点検状態・不備表示）

### 3.3 消火器登録・編集
- フィールド: 設置場所（必須）、製造者、型番、種別、消第、薬剤量、製造年、製造番号（必須）、備考
- ラベル撮影→AI自動入力→プリセットマッチング→フォーム入力
- プリセット未該当時にカスタムプリセット登録の確認

### 3.4 AI自動入力
- Gemini API (gemini-2.0-flash) でラベル写真から4項目抽出（sn, model, year, certNo）
- 最小プロンプトで安定出力。プリセットマッチングはアプリ側ロジック
- オフライン時は手動入力にフォールバック

### 3.5 点検記録
- 本体容器・圧力計・設置状況・表示ラベル（○△×／）
- 総合判定（○× 必須）、不備内容、措置
- 不具合時は報告カード画面に自動遷移

### 3.6 報告カード（看板方式）
- 不具合情報（写真＋施設名＋日時＋消火器情報＋不備内容＋措置）を1画面に看板レイアウト
- 不具合写真を撮影して看板内に表示
- Canvas APIで看板全体を画像化→IndexedDBに保存
- 画像1枚が自己説明的な報告書として機能する

### 3.7 データ出力
- 台帳CSV: 全消火器、紙台帳列順、BOM付きUTF-8
- 不具合報告xlsx: 報告カード画像を2列タイルで並べるだけ。ExcelJS dynamic import

### 3.8 設定・デバッグ
- Gemini APIキー管理
- カスタムプリセット一覧・削除
- バージョン表示、全データ削除、SWキャッシュ削除

## 4. 技術選定

| 技術 | 理由 |
|---|---|
| Vite + React | 既存スタック準拠 |
| localStorage | 台帳・点検・プリセット・設定 |
| IndexedDB | 報告カード画像（Canvas生成Blob） |
| Gemini API 無料枠 | ラベルOCR |
| Canvas API | 報告カード画像化。外部ライブラリ不要 |
| ExcelJS (dynamic import) | 週報xlsx |
| GitHub Pages | 無料デプロイ |
| PWA | オフライン・ホーム画面追加 |

## 5. 受入基準

1. GitHub Pagesでアクセスできること
2. ラベル撮影→AI自動入力→プリセットマッチ→保存がPixel 10で完了すること
3. カスタムプリセットが追加でき、以後ワンタップ登録できること
4. 点検で不具合→報告カード画面→写真撮影→画像保存ができること
5. 台帳CSVが正しく出力され、Excelで開けること
6. 週報xlsxに報告カード画像がタイル配置で出力されること
7. PWAインストール、オフライン操作（AI除く）ができること
8. gzip 200KB以下（ExcelJS除く）
